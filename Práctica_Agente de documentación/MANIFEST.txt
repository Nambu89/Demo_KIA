EJERCICIO 2: Documentation Agent KIA
═══════════════════════════════════════════════════════════════════════════════

Archivo: ejercicios/02_doc_agent/doc-generator.agent.md (17 KB)

Propósito:
  Crear un agent profesional y enterprise-ready de documentación

Funcionalidades (4 modos):
  1. Changelog Generator
     - Genera CHANGELOGs en formato "Keep a Changelog 1.0.0"
     - Lee git log
     - Categoriza por: Added, Changed, Deprecated, Removed, Fixed, Security
     - Enlaza a issues y PRs relevantes
     
  2. Code Documentation
     - Documenta funciones/clases/módulos
     - Genera docstrings Google Style (Python) o JSDoc (JS/TS)
     - Incluye: descripción, Args, Returns, Raises, Ejemplos
     - Formato: 100% copy-paste ready
     
  3. ADR Generator (Architecture Decision Records)
     - Crea ADRs siguiendo estándar oficial
     - Secciones: Contexto, Opciones, Decisión, Consecuencias
     - Incluye: métricas de éxito, plan de implementación
     - Formato: profesional y formal
     
  4. README Updates
     - Actualiza READMEs con estructura estándar
     - NO elimina contenido existente (quirúrgico)
     - Mantiene voz y tono original
     - Incluye: descripción, características, instalación, uso, testing

Tools utilizadas:
  - read (leer archivos)
  - editFiles (crear/modificar archivos)
  - search (buscar en código)
  - codebase (análisis de código)
  - runCommands (ejecutar git, npm, etc.)
  - changes (ver git diff)

Restricciones:
  ✗ NO puede modificar código fuente
  ✗ NO puede hacer git commit
  ✗ NO puede modificar tests
  ✓ Puede crear/modificar SOLO documentación

Invocaciones ejemplo:
  @Documentation Agent KIA: Genera el changelog de los últimos 15 commits
  @Documentation Agent KIA: Documenta el archivo src/api/handlers.py
  @Documentation Agent KIA: Crea un ADR para la migración a microservicios
  @Documentation Agent KIA: Actualiza el README con la estructura actual

Tiempo de enseñanza: 45 minutos
